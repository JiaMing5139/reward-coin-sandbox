# 激励广告金币定价 —— RL Sandbox 实验

用一个最小可运行的玩具系统，验证「强化学习能否学会激励广告的金币动态定价」。
纯 numpy 实现，无需 GPU / torch。

本系统是一个**序列决策**问题: 一个用户的一天是一条轨迹(多次广告请求)，金币不仅影响
当次利润，还通过**用户留存**影响后续能不能继续看广告。模型用 **TD/Q-learning
(Fitted Q-Iteration)** 学长期价值，决策受**日预算**约束；训练数据可切换**无偏/有偏**
两种来源，用来观察 off-policy 分布漂移的影响。

## 整体说明

这套实验有三个角色，先理解它们的关系，后面就都顺了:

- **Sandbox（真相 / 上帝视角的环境）**
  人为设定真实世界的运行规律: 金币如何影响用户观看、点击、竞胜、留存、平台利润。
  这些参数和公式是"客观真相"，但对模型**完全不可见**。它是一个**有状态环境**(标准 Gym 范式):
  `reset()` 开新的一天，`step(coins)` 推进一步并演化世界状态。
- **模型（尝试去理解环境的人）**
  模型看不到真相，只能拿到一堆 `(状态, 发的金币, 即时利润, 下一状态)` 的历史记录(transition)，
  用 TD/Q-learning 反推"从某状态发某金币，长期能赚多少"(即 Q 值)，进而决定最优金币。
- **对照基线（衡量模型学得好不好的标尺）**
  - `random`: 乱发金币，起点。
  - `behavior`: 采训练日志用的"线上行为策略"。
  - `myopic`(短视真值): **作弊基线**——每步直接用上帝的单步利润公式挑最优金币，但**完全不顾未来**。
    它单步完美、目光短浅。模型只有**打败 myopic**，才证明真的学到了"留住用户"的长期价值。

**三层抽象**(理解 Sandbox 的关键):

| 层 | 内容 | 可见性/演化 |
|---|---|---|
| **上帝参数** | 物理定律(金币→观看/点击/竞胜/留存的系数) | 全程不变，对模型不可见 |
| **World State** | 真正随 `step` 演化的动态量: `budget_left`/`steps_left`/`alive` + 当前用户 | 每步演化 |
| **Observation** | 模型看到的 16 维向量 = World State 的投影(用户+本次广告+预算+步数) | 模型输入 |

> 本次广告/上下文是每步**外生注入**的随机扰动(不受 action 影响)，不属于 World State。

**整体链路**:

```
 Sandbox(有状态真实环境, 真相不可见)
      │ reset() 开一天 → step(coins) 逐步推进, 含留存判定与预算扣减
      ▼
 transition 日志 (s, a, r, s', done)        ← 模型唯一能看到的东西
      │ 行为策略可选: 无偏(随机探索) / 有偏(线上策略, 制造覆盖空洞)
      ▼
 模型 MLP  ──Fitted Q-Iteration(TD)──▶  Q(s,a) ≈ 即时利润 + γ·未来最大Q
      │ 推理: 每步扫金币档(受预算约束), 选 Q 最大的
      ▼
 定价策略  ──rollout 评估──▶  放回真环境跑一天, 比每用户日累计利润
                              对照 random / behavior / myopic / model
```

一句话: **Sandbox 是真相，模型通过有限的 transition 日志去学长期 Q，
能否打败"短视真值(myopic)"是检验它有没有学到长期价值的标尺。**

## 背景问题

激励广告里，金币是给用户的激励。金币发多少是个权衡：

- **金币太少** → eCPM 低 → 竞不到曝光 → 没收入
- **金币太多** → 竞胜率高，但发金币的成本吃掉利润

所以存在一个**最优金币**，让平台利润最大。本项目用一个"上帝视角"的模拟器（Sandbox）定义真实世界，
让模型只通过观察回放日志去学这个最优定价，再和理论最优（oracle）对比，看它学得有多好。

项目还支持**两类广告**（自竞价 / 联盟），用来验证一个实际问题:
两类广告竞价逻辑不同，RL 模型只靠一个"广告类型标识"，能否自发学出差异化的金币策略。

## 建模框架

| 要素              | 内容                                                          |
| --------------- | ----------------------------------------------------------- |
| **State (obs)** | 用户 + 本次广告 + 上下文 + 广告类型 + **预算/步数**，共 16 维                    |
| **Action**      | 单条广告发放的金币数 (10 档: 0,2,4,...,18)                             |
| **即时 Reward**   | 单步平台利润 = 平台收入 − 金币成本                                        |
| **优化目标**        | **折扣累计利润** Σ γᵗ·rₜ (γ=0.9)，即长期价值而非单步                        |
| **Episode**     | 一个用户的一天: 多次广告请求，直到不留存 / 步数耗尽 / 预算耗尽                         |

**因果链**（金币如何影响利润）:

```
                    ┌─► 用户观看意愿 ┐
金币 ──────────────┤                ├─► 曝光(用户看 且 竞胜) ─► 收入
                    └─► pCTR ─► eCPM ─► 竞胜 ┘
金币 ─────────────────────────────────────► 曝光后发金币 = 成本
金币 ─────────────► 用户留存(下一步还来) ──► 未来更多曝光机会(长期价值)
```

金币有三条**收益**路径(观看、竞胜、**留存**)和一条**成本**路径。
其中"留存"是**长期价值**的来源: 现在多发金币 → 留住用户 → 后面几步还能赚 →
所以最优金币会**高于只看单步的短视策略**(这正是序列建模的意义)。

**两类广告的差异**（核心实验点之一）:

| <br /> | A 类(自竞价)           | B 类(联盟)                      |
| ------ | ------------------ | ---------------------------- |
| eCPM   | `bid × pCTR`，金币能抬高 | 外部渠道返回 `g_ext_ecpm`，**金币无关** |
| 竞胜率    | 随金币↑               | 与金币**无关**(外生 eCPM 固定)        |
| 平台收入   | 二价: 次高位 eCPM(点击计费) | 外生 eCPM(曝光结算)                |
| 金币的收益  | 观看 + 竞胜 + 留存       | 观看 + 留存(无竞胜增益)               |

## Sandbox 详解

### 上帝参数（`sandbox.py` 中 `Sandbox` 类的类属性，对模型不可见）

| 参数               | 默认值  | 含义                   |
| ---------------- | ---- | -------------------- |
| `COIN_TO_CTR`    | 0.06 | 金币对点击率(logit)的提升     |
| `LTV_TO_CTR`     | 0.40 | 用户价值对点击的影响           |
| `GQ_TO_CTR`      | 6.0  | 广告点击质量权重             |
| `CTR_BIAS`       | -2.2 | 点击率偏置                |
| `WIN_SHARP`      | 1.0  | 竞胜 sigmoid 陡峭度       |
| `THRESH_BASE`    | 6.0  | 竞争水位基线(也是二价成交价)      |
| `THRESH_PEAK`    | 2.0  | 晚高峰(20点)水位凸起         |
| `COIN_COST`      | 0.5  | 金币面值→真实成本折算          |
| `WATCH_BIAS`     | -0.5 | 观看意愿偏置(即时)           |
| `COIN_TO_WATCH`  | 0.22 | 金币对观看意愿的提升(即时)       |
| `SENS_TO_WATCH`  | 2.0  | 用户敏感度对观看的负作用         |
| `RETAIN_BIAS`    | -0.6 | 基础留存(低→不发金币就易流失)     |
| `COIN_TO_RETAIN` | 0.28 | **金币对留存的提升(长期价值来源)** |
| `SENS_TO_RETAIN` | 0.8  | 敏感用户更易流失             |

### 配置开关（`sandbox.py` 顶部 `CONFIG` 字典，一处改全局生效）

| 键                 | 默认    | 含义                                  |
| ----------------- | ----- | ----------------------------------- |
| `union_enabled`   | True  | 是否接入联盟广告(False=只有自竞价 A 类)           |
| `union_ratio`     | 0.30  | 联盟广告占比                              |
| `union_ecpm_lo/hi`| 5/11  | 联盟广告外生 eCPM 范围                      |
| `horizon`         | 6     | 一个用户日最多几次广告请求                       |
| `gamma`           | 0.9   | 折扣因子(未来利润打折)                        |
| `day_budget`      | 30.0  | 一个用户日的金币预算                          |
| `behavior_biased` | False | 训练日志来源: False=无偏随机; **True=有偏(演示 off-policy)** |

### 核心接口（标准 Gym 范式，向量化并行 N 个用户日）

| 方法                       | 作用                                    |
| ------------------------ | ------------------------------------- |
| **`reset(n)`**           | 开 n 个新用户日，返回首个 observation 矩阵         |
| **`step(coins)`**        | 世界并行推进一步，返回 `(obs, reward, done, info)` |
| `retain_prob(f, coins)`  | 留存概率(金币换"下一步还来"，长期价值来源)              |
| `immediate_profit(f, c)` | 单步期望利润(给 myopic 短视基线用)               |
| `watch_prob/pctr/ecpm/win_prob` | 因果链内部各环节概率                     |

`step()` 内部演化 World State: 扣预算 → 减步数 → 掷留存 → 判 `done` → 切换下一条外生广告。

## 文件结构

| 文件           | 职责                                                       | 含 main |
| ------------ | -------------------------------------------------------- | ------ |
| `sandbox.py` | 有状态真实环境: 三层抽象 + `reset()`/`step()` + 留存/预算 + `CONFIG`    | 否      |
| `model.py`   | 策略模型: numpy 手写 MLP (He 初始化 + Adam + 反向传播)，学长期 Q         | 否      |
| `main.py`    | 主流程: 采 transition → FQI(TD) 训练 → rollout 评估，逐步注释         | 是      |

## 怎么用

```bash
cd reward-coin-sandbox
python3 main.py
```

主流程 5 步 (见 `main.py` 注释):

1. **采集回放日志**: 行为策略跑 episode，记录 `(s, a, r, s', done)` transition
2. **特征工程**: 标准化 obs + 拼接 `coins`、`coins²`
3. **训练 Q 模型**: Fitted Q-Iteration(TD)，目标 = `即时利润 + γ·下一状态最大Q`(自举)
4. **推理**: 每步扫金币档(受预算约束)，取预测 Q 最大者
5. **rollout 评估**: 各策略放回真环境跑一天，比每用户日累计利润

### 想改实验

- **开关 off-policy**: 改 `CONFIG["behavior_biased"]`(False=无偏 / True=有偏)，看模型是否退化
- **开关联盟广告**: 改 `CONFIG["union_enabled"]`、`union_ratio`
- **改长期权重**: 调 `COIN_TO_RETAIN`/`RETAIN_BIAS`(留存价值)、`gamma`(折扣)
- **改预算松紧**: 调 `CONFIG["day_budget"]`(调小→预算约束更绑定)
- **改环境难度**: 调上帝参数 (`COIN_COST`、`THRESH_BASE` 等)

> 试试: 把 `behavior_biased` 设 True，model 会从"超越短视 +0.22"跌到"不如短视 −0.15"
> ——这复现了 off-policy 离线 RL 最经典的坑: 有偏日志的覆盖空洞让 Q 在分布外瞎猜。

## 名词解释

- **Sandbox(上帝视角环境)**
  人为设定真实因果规则的**有状态**模拟器。`reset(n)` 开 n 个用户日，`step(coins)` 推进一步。
  参数对模型不可见，模型只能从 `step` 吐出的反馈学习。
- **三层抽象**: 上帝参数(定律,不变) / World State(预算·步数·存活,随 step 演化) /
  Observation(模型看到的 16 维投影)。本次广告是每步外生注入，不算 World State。
- **Episode(用户日 / 轨迹)**
  一个用户的一天，含多次广告请求。直到不留存 / 步数耗尽 / 预算耗尽才结束。
- **transition (s, a, r, s', done)**
  训练样本的形式: 状态 s 下发金币 a，得即时利润 r，转移到 s'，是否结束 done。
  TD/Q-learning 直接用这种四元组学习(不需要整条轨迹的累计回报)。
- **即时利润 vs 折扣累计回报**
  即时利润 = 单步赚的钱。优化目标是**折扣累计** Σγᵗrₜ，即把未来的钱打折加进来 → 长期价值。
- **留存(retain_prob)**
  看完这条后继续留下看下一条的概率。金币越多越愿留 → 这是**长期价值的来源**:
  现在多发金币留住用户，换后面几步还能赚。没有留存这条链，序列建模就没意义。
- **myopic(短视真值, 关键对照)**
  作弊基线: 每步直接用上帝单步利润公式挑最优金币，但**完全不顾未来**。单步完美、目光短浅。
  模型**打败 myopic** 才证明学到了长期价值(本项目 model 比 myopic 高 +0.22)。
- **Fitted Q-Iteration (TD/Q-learning)**
  训练方法: 目标 = `即时利润 + γ·下一状态的最大Q`，未来项用**上一轮模型自己估**(自举)。
  相比"用整条轨迹的蒙特卡洛回报当目标"，TD 的目标方差小得多，模型才学得动。
- **预算(pacing)**
  每个用户日有金币预算(`budget_left` 写在 obs 里)，发完没了。逼策略把金币留给高价值请求。
- **on-policy vs off-policy**
  - on-policy: 训练数据必须来自当前策略本身，策略一更新旧数据就作废(强依赖采样过程)。
  - off-policy(本项目): 学 `Q(s,a)`(环境客观属性)，可用任何来源的日志。**但前提是数据要覆盖
    到关心的 (s,a)**；若行为策略有偏(`behavior_biased=True`)制造覆盖空洞，模型在分布外就瞎猜。
- **联盟广告 vs 自竞价广告**
  - 自竞价(A): eCPM=出价×pCTR，金币能抬高 eCPM 进而提竞胜率。
  - 联盟(B): eCPM 由外部渠道返回，金币**不参与竞价排序**，只在胜出曝光后影响成本。
  - 由 `CONFIG["union_enabled"]` 开关。验证"只加一个 `ad_type` 标识，模型能否自发适配差异"。
- **rollout 评估**
  把策略放回真环境跑完整 episode，统计每用户日累计利润。序列+预算+随机下没有精确 oracle，
  用 rollout 比各策略(random/behavior/myopic/model)的真实表现。

## 大致结论

各策略 rollout 每用户日累计利润 (联盟广告占比 30%，预算 30):

```
                    无偏日志        有偏日志(off-policy)
random              1.39           1.39
behavior(线上)      1.37           2.90
myopic(短视真值)    3.15           3.15
model               3.36  ✅        3.00  ❌
--------------------------------------------------
model vs myopic     +0.22(更优)     -0.15(退化)
```

1. **模型学会了长期价值，打败了短视真值(myopic)**——
   在无偏数据下，model(3.36) > myopic(3.15)，提升 +0.22。myopic 每步单步完美但不顾未来，
   model 则学会了"现在多发点金币留住用户、换后面几步的利润"。这正是序列建模(留存→长期回报)
   的意义: 短视策略结构上拿不到这部分钱。

2. **TD/Q-learning 是学得动的关键**——
   一开始用蒙特卡洛(整条轨迹累计回报)当训练目标，方差极大(目标范围 [-13, +47])，
   loss 卡在 0.93 降不动，model 反而不如 myopic。换成 Fitted Q-Iteration(目标=即时利润
   + γ·下一状态最大Q，未来项自举)后方差骤降，TD 目标逐轮抬升收敛，model 才真正学好。

3. **off-policy 分布漂移会让模型退化**——
   把训练日志从"无偏随机探索"换成"有偏的线上策略"(`behavior_biased=True`)，
   model 从 +0.22 跌到 -0.15，且 TD 拟合 MSE 后期飙升(覆盖空洞导致自举不稳)。
   这复现了离线 RL 最经典的坑: **off-policy 能复用日志，但数据没覆盖到的 (状态,金币) 上，
   Q 只能瞎猜**——正是真实业务里"离线训练好、上线翻车"的一大根源。

4. **预算约束塑造决策**——
   obs 含 `budget_left`/`steps_left`，策略推理时受预算上限约束，rollout 真实绑定预算。
   模型据此学会在有限预算下做跨请求的分配。

5. **联盟广告: 只加 `ad_type` 标识即可自发适配**——
   两类广告竞价逻辑不同，但只要 reward 用真实回放、样本充足、占比够大，
   模型靠一个类型标识就能学出差异化策略；占比过小则会被主类淹没。

   **那为什么有人担心"两类广告共用模型会离线/线上不一致"?** 这种担心对了一半:
   - ❌ **误解的那半**: 以为"加个标识、共用模型"本身会导致不一致。不会——
     只要离线 reward 用的是线上真实回放，"逻辑不同"只是让模型多学一套响应，不制造不一致。
   - ✅ **真实风险一(致命)**: 若离线 reward 不是真实回放，而是用一个**竞价模拟器**算的，
     而那个模拟器默认"金币↑→eCPM↑→竞胜↑"(只对自竞价成立)，套到联盟广告上就错了：
     离线以为"给联盟广告多发金币能提竞胜多赚"→ 学会多发；线上联盟竞胜率根本不动 → 纯亏成本。
     这才是真正的离线/线上不一致，**罪魁是模拟器用错竞价假设，加不加标识都救不了**。
   - ✅ **真实风险二(覆盖)**: 联盟广告样本太少 → 模型对 B 类没学好 → 离线指标(被 A 类主导)好看、
     线上 B 类流量翻车。根源是样本覆盖不足，不是逻辑冲突(调小 `union_ratio` 可复现)。

   结论: 正确应对不是纠结"要不要加标识"，而是**确保离线 reward 对两类用正确竞价逻辑
   (或直接用真实回放) + 保证联盟广告样本充足**。本项目用真实 `step()` 回填 reward，
   规避了风险一，故加一个标识就够。

> 注: 这是用于直观理解的玩具系统，Sandbox 的因果规则是人为设定的。
> 真实业务中环境未知、非平稳，奖励有延迟/归因噪声，且 off-policy 问题更严重，
> 不能直接照搬结论。本项目的价值在于把这些"坑"用最小代码可复现地演示出来。
